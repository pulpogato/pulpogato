## 2025-05-14 - Failed optimization of union-type deserialization using convertValue
**Learning:** Attempted to replace JSON string round-trips with `ObjectMapper.convertValue` in `FancyDeserializerSupport` to improve performance. However, this caused regressions in union types like `SingularOrPlural` and `StringOrInteger`. Direct object-to-object conversion in Jackson behaves differently than JSON serialization/deserialization regarding type coercion (e.g., converting Numbers to Strings). The existing tests rely on the loose coercion provided by the JSON parsing path.
**Action:** Prefer JSON string round-trips for union type resolution when loose coercion is required by the model. Avoid direct `convertValue` for these specific generic containers unless tests are updated to support stricter type matching.
